Personas and Agents

A generative agent is built in two layers. The persona is its fixed identity — text that is legible to a language model. The mind wraps that persona around a memory stream, a plan, and a status, and is what every cognitive function reads and writes. The agent is the Agents.jl body that carries a mind through a simulation.

The split between mind and body is deliberate: the cognitive layer dispatches on the mind accessor, never on a concrete agent type, so the same machinery serves the ready-made non-spatial GenerativeAgent and any spatial @agent type with a mind field equally well.

Personas

A Persona is the part of an agent that does not change during a run. It is rendered into every cognitive prompt by describe.

GABM.Persona — Type
Persona(name; age, occupation, traits, background)

The natural-language identity of a generative agent.

A persona is the part of an agent that does not change during a simulation: a stable description of who the agent is, as opposed to the MemoryStream that records what has happened to it. It is rendered into every prompt the agent's mind issues, via describe.

Arguments

  • name::AbstractString: the agent's name (positional, required).
  • age: an Integer, or nothing if unspecified.
  • occupation::AbstractString: a short role description.
  • traits: an iterable of short trait strings (e.g. ["curious", "frugal"]).
  • background::AbstractString: one or more sentences of free-text history.

Example

isabella = Persona("Isabella Rodriguez";
    age = 34,
    occupation = "café owner",
    traits = ["warm", "organised", "ambitious"],
    background = "Runs Hobbs Cafe and is planning a Valentine's Day party.")
source
GABM.describe — Function
describe(p::Persona) -> String

Render a persona as a compact, prompt-ready identity block.

Only the fields that were supplied appear, each on its own line, so the result stays readable whether a persona was specified richly or minimally. This is the exact text that the cognitive layer (decide, reflect!, plan!) places at the head of every prompt.

Example

julia> println(describe(Persona("Ada"; occupation = "engineer")))
Name: Ada
Occupation: engineer

Minds

A Mind is the cognitive state of an agent: a persona, a MemoryStream, a plan, and a status line.

GABM.Mind — Type
Mind(persona; memory, plan, status)

The cognitive state of a generative agent.

A Mind bundles the four things every cognitive function in GABM.jl needs:

  • persona::Persona — the agent's fixed identity (see Persona).
  • memory::MemoryStream — its evolving record of experience.
  • plan::Vector{String} — its current intentions, as set by plan!.
  • status::String — a one-line description of what it is doing right now.

The split between Mind and the agent body is deliberate. A Mind carries everything the language model reasons about; the body carries everything the spatial or network model needs. Keeping them in separate objects lets the same cognitive machinery serve a non-spatial GenerativeAgent, a grid agent, or a node in a social network without change.

Agents

GenerativeAgent is the ready-made body for non-spatial models. For a spatial model, define an agent with Agents.@agent that has a mind field; the mind and persona accessors then apply to it automatically.

GABM.GenerativeAgent — Type
@agent struct GenerativeAgent(NoSpaceAgent)
    mind::Mind
end

A ready-made, non-spatial generative agent.

GenerativeAgent is an Agents.jl agent with two fields: the id supplied by NoSpaceAgent, and a Mind. It is the agent type used by generative_abm unless another is requested, and it covers every model in which agents have no position — conversations, markets, committees, opinion dynamics.

For a spatial model, define an agent with a mind field instead, e.g.

using Agents, GABM
@agent struct Resident(GridAgent{2})
    mind::Mind
end

Every cognitive function (observe!, decide, …) accepts such an agent directly, because they dispatch on the mind accessor rather than on the concrete agent type.

source
GABM.mind — Function
mind(agent) -> Mind

Return the Mind of agent.

The fallback implementation returns agent.mind, so any Agents.jl agent with a mind field works with GABM.jl out of the box. The cognitive layer is written entirely against this accessor; a modeller whose agent stores its mind elsewhere need only add a method to mind.

persona(x) -> Persona
